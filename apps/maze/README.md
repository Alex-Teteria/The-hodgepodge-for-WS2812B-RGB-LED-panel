# LED Maze (MicroPython, RP2040, NeoPixel 16×16)

Проєкт генерує лабіринт на ґратці 16×16, будує граф прохідних клітин і візуалізує:

- DFS-прохід із поверненнями назад (анімація “переміщення” по лабіринту)
- BFS-найкоротший шлях між двома точками
- “Доганялки” (одна точка тікає, інша переслідує)

Платформа: RP2040 + MicroPython  
Вивід: NeoPixel/WS2812 LED-матриця 16×16

---

## Залежності

- MicroPython (RP2040)
- модулі: neopixel, time, random

Примітка:  у цьому проєкті черга реалізована як **list + head** (без deque).

---

## Координати та індексація LED

Координати клітин: (i, j), де:

- i — рядок (0..n-1)
- j — колонка (0..m-1)

Відображення координат у NeoPixel-індекс (з урахуванням “змійки”):

    def coord_to_pix(i, j):
        return m * i + j if i % 2 else m - j - 1 + m * i

## Ключові ідеї та інваріанти

### Граф прохідних клітин

- Вершини — всі клітини, які НЕ є стіною.
- Ребро існує між двома вершинами, якщо вони сусіди по 4-напрямках (вгору/вниз/ліворуч/праворуч).

Умова суміжності (без квадратного кореня):

- dist = (x1-x2)**2 + (y1-y2)**2
- суміжні, якщо dist == 1

### Інваріанти генератора лабіринту `maze_generator.py`

### Спосіб побудови лабіринта

1. Будуються стіни як суцільні горизонтальні та вертикальні лінії (плюс рамка),
   товщина стіни — 1 клітинка.
   Це розбиває поле на “кімнати/комірки” (прямокутні області прохідних клітин),
   які утворюють зв’язну ґратку по сусідству (кімната має сусідів через спільну стіну).

2. Випадковим чином, утворюємо “дірки” у стінах → це об’єднує деякі кімнати в більші компоненти.

3. Будуємо граф з точoк, що не увійшли до стін лабіринту

4. Пошук компонент зв’язності графа який утворився.

5. Будуємо граф між компонентами зв'язності, де
    вершини - компоненти
    ребра - коли відстань між компонентами = 2 (коли якісь вершини торкаються кутами), або = 4, коли вершини розміщені через клітинку 

6. Будуємо остовне дерево, яке повинно охопити усі компоненти.

7. Пробиваємо перегородки в стінах в місцях ребер остовного дерева.

8. Створюємо вхід та вихід з лабіринта

Ключове для п.5: для будь-яких двох “сусідніх кімнат” між ними є стіна товщиною 1,
   отже існують дві прохідні клітинки по різні боки стіни на відстані dist==4
   (або dist==2 для кутового випадку), і тому таке ребро між ними завжди існує.
   Тому граф компонент буде зв’язним (один компонент зв'язності).  

Гарантія (для поточного генератора): граф компонент зв’язний, отже остовне дерево охоплює всі компоненти, і після видалення стін в місцях ребер лабіринт стає прохідним.

Якщо змінювати генератор (товщина стін, правила суміжності/пробиття), ці умови треба перевірити заново.

## Опис файлів

### graph.py

Клас `Graph` для роботи з графом.

Ключові методи:

- build_edges_and_adjacency() — швидке O(V) створення edges і G
- dfs(start) — DFS як parent-map (дерево DFS)
- dfs_walk_edges(start, finish=None) — DFS як маршрут руху (ребра з поверненнями)
- bfs(start) — BFS як parent-map
- bfs_distance(start) — BFS відстаней + prev
- find_path(start, finish, prev=None) — відновлення найкоротшого шляху
- dfs_tree(start) — остовне дерево 
- find_components(method='dfs'|'UnionFind') — компоненти зв’язності

Примітка: порядок обходу DFS може “плавати”, бо G[v] — set. Якщо потрібен стабільний маршрут — необхідно задати фіксований порядок сусідів.

### maze_generator.py

Генератор лабіринтів:

- build_maze(...) — створює лабіринт (стіни), пробиває “перемички” для зв’язності, опційно створює start/finish
- допоміжні: build_grid, build_wall, build_edges, break_wall, create_start_finish, build_border

### maze_dfs.py

DFS-візуалізація:

- генерує лабіринт
- будує Graph(vertices)
- викликає dfs_walk_edges(start, end) і малює шлях на LED

Це не найкоротший шлях, а “прогулянка DFS” з поверненнями.

### maze_bfs.py

BFS-найкоротший шлях:

- генерує лабіринт
- випадковий вибір start/end 
- BFS + відновлення маршруту start -> end як список ребер
- візуалізація на LED

### maze_bfs_bfs.py

Візуалізація “Доганялки”:

- точка 1 (втікає) обирає далекі цілі на основі BFS-відстаней
- точка 2 (наздоганяє) йде найкоротшим шляхом використовуючи алгоритм BFS
- таймери змінюють діапазон випадкового вибору, щоб уникати циклів

---

## Запуск

1) DFS-візуалізація: запустіть `maze_dfs.py`  
2) BFS-найкоротший шлях: запустіть `maze_bfs.py`  
3) Доганялки: запустіть `maze_bfs_bfs.py`
