# switcher для застосунків (RP2040 / MicroPython)

Утиліта `switcher` реалізує перемикання між застосунками з каталогу `apps/` по натисканню кнопки. Поточний індекс застосунку зберігається у файлі `app_idx.txt`. Після підтвердженого натискання індекс збільшується і виконується `machine.reset()`.

---

## Структура проєкту

Рекомендована структура:
/  
├─ main.py  
├─ switcher.py  
├─ app_idx.txt (створюється/оновлюється автоматично)  
└─ apps/  
&emsp;&emsp;├─ app01.py  
&emsp;&emsp;├─ app02.py  
&emsp;&emsp;└─ ...  

---

## Вимоги

- Плата: RP2040 (або інша, де доступні `machine.Pin`, `machine.reset`).
- Прошивка: MicroPython.

---

## Як це працює

1. На старті `switcher` сканує каталог `apps/` і формує список застосунків `*.py` (який відсортовує за назвою).
2. Поточний індекс читається з `app_idx.txt` (за відсутності/помилці — `0`).
3. Кнопка підключена як `PULL_UP` і генерує IRQ по спаду (`IRQ_FALLING`).
4. IRQ-обробник не робить I/O — лише ставить прапорець `_pending` і фіксує час.
5. `switcher.service()` (функція, яку треба викликати з застосунку) виконує debounce та, якщо натискання підтверджене:
   - оновлює `app_idx.txt`;
   - викликає `machine.reset()`.

---

## Підключення кнопки

- Кнопка: між GPIO `BTN_PIN` та GND.
- Внутрішня підтяжка вгору (`PULL_UP`) увімкнена в коді.

---

## Налаштування

У `switcher.py`:

- `APPS_DIR = "apps"` — каталог із застосунками.
- `IDX_FILE = "app_idx.txt"` — файл індексу.
- `BTN_PIN = 15` — GPIO кнопки.
- `DEBOUNCE_MS = 40` — debounce у мс.

---

## main.py (лоадер)

Приклад `main.py`, який виконує поточний застосунок як `__main__` і передає йому доступ до `switcher`:

```python
# main.py
import switcher

path = switcher.current_app_path()

with open(path, "r") as f:
    code = f.read()

# Дамо програмі доступ до switcher
g = {"__name__": "__main__", "switcher": switcher}
exec(code, g, g)
```

## Вимоги до інтеграції в застосунках (обов’язкові вставки)

1) Регулярно викликати `switcher.service()`

Вставити у “часто виконуване місце” (кадр/ітерація/петля), щоб кнопка оброблялась без затримок.

2) Заміна блокуючих затримок

Замість `time.sleep_ms(...)` використовувати `switcher.sleep_ms(...)`, щоб під час затримок також оброблялась кнопка.  
> **Примітка:** якщо цикл гарантовано завжди містить `switcher.sleep_ms(...)` і “кадр” короткий, можна обійтись без `switcher.service()` у циклі. Але якщо є довгі обчислення/рендер/IO між sleep — `service()` у циклі потрібен.  

## Мінімальний шаблон застосунку

Варіант без import switcher (бо switcher передається через main.py):  

```python
import time

def main():
    while True:
        # 1) обов’язково: обслуговування кнопки
        switcher.service()

        # ... логіка / рендер / робота з периферією ...

        # 2) обов’язково для затримок: заміна sleep
        switcher.sleep_ms(20)

main()
```

> **Примітка:** якщо цикл гарантовано завжди містить `switcher.sleep_ms(...)` і “кадр” короткий, можна обійтись без `switcher.service()` у циклі. Але якщо є довгі обчислення/рендер/IO між sleep — `service()` у циклі потрібен.  

## Додавання застосунку

1. Додайте файл `apps/your_app.py`.
2. Назва файлу впливає на порядок (список сортується).
3. Перезапустіть плату (або натисніть кнопку перемикання) — список `APPS` буде  сформовано заново при імпорті.

## Обмеження та очікувана поведінка

- Короткі натискання можуть ігноруватись (через `DEBOUNCE_MS` і подальшою перевіркою та підтвердженням стану кнопки).
- Перемикання реалізоване через `machine.reset()`, тому виконання поточного коду після успішного натискання переривається.
- Якщо застосунок довго блокує виконання `service()` / `switcher.sleep_ms(...)`, реакція на кнопку буде затримуватись.  

## Діагностика типових проблем

- `No apps found in /apps`
  Перевірте, що існує каталог `apps/` і в ньому є `*.py`.

- Кнопка не перемикає
    Перевірте:
  
  - правильний `BTN_PIN`;
  - підключення кнопки `GPIO`↔`GND`;
  - що застосунок викликає `switcher.service()` і/або використовує `switcher.sleep_ms()` замість `time.sleep_ms()`.

- Індекс “не зберігається”  
  Перевірте можливість запису у файлову систему та наявність `app_idx.txt`.
