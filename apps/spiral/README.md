# Спіральна візуалізація на LED-матриці (NeoPixel / WS2812, MicroPython)

## Опис

Скрипт для MicroPython, який будує спіральний порядок обходу прямокутної LED-матриці `n×m` і відтворює анімацію:

1) “скручування” (заповнення) спіралі одним кольором,
2) “розкручування” (повторний прохід у зворотному напрямку) іншим кольором.

## Обладнання та вимоги

- Плата з MicroPython (RP2040 / ESP32 тощо)
- LED-матриця на WS2812/NeoPixel (у прикладі 16×16 = 256 світлодіодів)
- Бібліотека `neopixel` (в MicroPython зазвичай доступна)

## Підключення

- Сигнальний вхід матриці → GPIO `neo_pin` (за замовчуванням `20`)
- Живлення матриці відповідно до її вимог (часто 5V) + спільна земля (GND) з платою 

## Налаштування

Основні параметри на початку файла:

- `n`, `m` — розмір матриці
- `neo_pin` — GPIO для керування LED
- `speed` — затримка в мс між кроками (один крок = оновлення 1 нового пікселя + `pix.write()`)

Палітра:

- `colors = (green, red, yellow, magenta, blue, nothing)`

Зауваження щодо “змійки” (serpentine):

- Функція `coord_to_pix(i, j)` реалізує відповідність координат матриці до індексу NeoPixel для конкретного фізичного підключення.
- У поточній формулі рядки чергують напрямок, при цьому парні рядки йдуть у зворотному напрямку. Якщо матриця підключена інакше — змінюйте `coord_to_pix`.

## Як формується спіраль

`find_path(n, m, rotation)` генерує список індексів (довжина `n*m`), що відповідає спіральному обходу:

- `rotation='right'`: порядок поворотів “вправо → вниз → вліво → вгору”
- `rotation='left'`: порядок поворотів “вниз → вправо → вгору → вліво”

Алгоритм є конструктивним: рухається в поточному напрямку, доки не “впирається” в межу / вже відвідану клітинку, після чого напрямок циклічно змінюється.

## Як відтворюється анімація

У `main_run(path_l, path_r, colors, color_2)`:

- обирається `color_1` (не дорівнює попередньому `color_2`)
- обирається новий `color_2` (не дорівнює ні `color_1`, ні попередньому)
- прохід по `path_r`: по одному пікселю, `pix.write()` кожного кроку
- прохід по `path_l[::-1]`: по одному пікселю у зворотному порядку

Це дає ефект “скручування” та “розкручування”.

## Продуктивність

- Побудова шляхів `find_path()` виконується один раз, складність `O(n*m)`.
- Основний час у циклі анімації витрачається на `pix.write()` (передача всього буфера WS2812) та `sleep_ms(speed)`.
- Регулювання швидкості виконується параметром `speed`.
